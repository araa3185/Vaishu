Overview

NSX reproduces the entire network model in software, enabling any network topology—from simple to complex multi-tier networks—to be created and provisioned in seconds.

NSX Intelligence - “Next Billion”
Every business has a story to tell and these days that story usually involves data. At a time when people expect software to help make sense of their network topology and services, NSX Intelligence was set out to strive data intelligence as the name says it all auto-detect issues backed up with ML powered resolution, visualize the network in a seamlessly intelligent way.

 

Requirement

NSX Intelligence - On a high level  Monitoring and Troubleshooting are the key that unfolds the broad spectrum of this feature

Monitoring broadly covers four different categories:

  • Network Performance Monitoring - To be able to detect changes in network performance over a period of time and across different grouping dimensions and trigger alerts based on user defined thresholds I.e data sources, system metrics, flow data and context data 
  • Network Connectivity - Network connectivity issues caused by configuration errors are often difficult to identify. It can be caused by several reasons, some of which are :
    1. Configuration errors
    2. Incorrect FW policies
    3. Underlying physical network issues  The goal of connectivity monitoring is to be able to detect network outages and help admin root cause the reason for outage
  • Application Monitoring - Monitoring the applications such as WebApp, DB and other services running in the VM with respect to Network
  • Security Monitoring - Monitor and validate whether the configured policies on NSX are working as expected using the flow and data source. Analyze flows that deviate from configured FW policies through probe functionality and generate traffic to validate distributed FW rules.

 

Troubleshooting:

NSX Intelligence works on systematic approach through user set guidelines and ML when it comes to troubleshooting. Based on specific symptoms NSXi will go through systematic elimination of potential problem and narrow down the root cause for the identified issue.

 

Problem

People working in modern day virtualization require more than just static charts and pretty looking visualizations; they need to go beyond by drilling deeper, probing further, and understanding the forces that influence their business network & security.

To begin with, It is very hard for customers to visualize their network setup. Most of NSX UI is based on data grids that are not conducive of clearly articulating how the network setup is related and troubleshooting was priority.

 

My Role

As a Senior designer, my role was to be the design stakeholder to ensure that data presentation is meaningful, compelling, and most importantly, easy to interpret. I laid the foundation for visualization architecture to be first of its kind in NSX through NSX Intelligence.

Defined the Interaction principles working with Customers and Product Stakeholders. Vitalized the product strategy and advocated user-centered data visualization to Engineers. Played a phenomenal role in scaling up the design team from 3 to 10 Member Team till date. Presented UX walkthrough to customers in VMworld and conducted usability sessions through focus groups involving end users. In the entire product journey design process was formalized at every step starting from research, idea conceptualization, validation through usability testing, mapping to business goals making it to be a market fit.

Challenge

Need an experience that is as immersive as possible, while making it easy for user to uncover deeper insights and drill deeper into their data in order to make better business decisions.

Approach

Users have data, they have questions, and they need an intelligence through visualization that will help them make sense of their data and turn it into useful business insights, while reducing uncertainty.

Persona

Over 1 million admin users, with 35 million active consultants across widely spread locations. We had very limited access to the users in different regions like US, EMEA, APAC.

Was given a choice to grasp user needs, but with no access to them directly. So captured user preferences expressed through interest graphs and surveys that emerged a user persona.

User Journey

Customers spend many hours and resources to create topologies and follow the primitive methods to analyze the network configuration through static data grid design. Expectation was a robust visualization with time machine that keeps them well informed through intuitive communication about their network configuration and security
